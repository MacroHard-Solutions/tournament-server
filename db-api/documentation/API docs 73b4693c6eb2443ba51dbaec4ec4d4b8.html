User table functions

Root link

Signup check on username —> Checks if the username doesn’t already exist 202 means it is acceptable 409 means there is an existing user with the username

curl --request POST \
  --url https://tournament-server.herokuapp.com/api/v2/user/signupCheck \
  --header 'Content-Type: application/json' \
  --data '{
  "username": "Wooden419"
}'

Example response

{
  "status": "OK",
  "message": "You may use this username"
}

User signup —> Adds user to database 201 means the object is created

curl --request POST \
  --url https://tournament-server.herokuapp.com/api/v2/user \

  "fName": "Katelynn",
  "lName": "Yost",
  "username": "multi-byte97",
  "userEmail": "Ismael.Walsh@example.org",
  "userPass": "gr2nlb08zr",
  "isAdmin": "false",
  "wantsNotifications": "false"
}'

Example response

{
  "status": "success",
  "message": "The user has been successfully inserted"
}

User login —> Checks user credentials 200 returns the object in user header if it is valid 404 indicates the user is not on the database

curl --request POST \
  --url https://tournament-server.herokuapp.com/api/v2/user/login \

Game table functions

Root link

Add game —> Adds a new Game to the table 201 means the new object is created

curl --request POST \
  --url https://tournament-server.herokuapp.com/api/v2/game/ \
  --header 'Content-Type: application/json' \
  --data '{
  "gameName" : "voluptatum-quaerat-officiis",
  "fileName" : "https://audie.info",
  "gameIcon" : "http://lorempixel.com/640/480/abstract"
}'

Example response

{
  "status": "success",
  "message": "Game added successfully",
  "newGame": [
    {
      "GAME_ID": "37f11817-c32a-11ec-a02e-0ab3cd6d5505",
      "GAME_NAME": "sint-laudantium-impedit",
      "FILE_NAME": "http://gail.name",
      "GAME_ICON": "http://lorempixel.com/640/480/abstract"
    }
  ]
}
